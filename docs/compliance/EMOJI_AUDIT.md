## Priority Tiers

Entries in `DefaultReplacements()` are grouped into four priority tiers based
on how commonly an emoji appears in practice and by what generates it.

| Tier | Label | Meaning |
|------|-------|---------|
| P0 | Critical | Default output of AI coding agents (Copilot, Claude) with no configuration. Must be substituted for clean text pipelines. |
| P1 | High | Established conventions: gitmoji commit messages, changelog tooling, README badges. Generated by tooling or strong community convention. |
| P2 | Medium | Platform/ecosystem indicators, CI dashboards. Commonly seen but not generated by agents by default. |
| P3 | Low | Decorative or domain-specific. Present for completeness; rarely appears in technical prose generated by agents. |

## Maintenance Workflow

### How to search for missing emoji

1. **Observe gaps in output** — Run `go test -run TestRepo` after processing
   any AI-generated file; the test suite enforces emoji-clean docs. If
   `ScanDir` reports a hit in a file that should be clean, the offending
   codepoint is an unsubstituted P0 candidate.

2. **Cross-reference gitmoji** — Check [gitmoji.dev](https://gitmoji.dev/) for
   new additions. Each release of gitmoji adds emoji that AI agents will start
   generating. Map the Unicode codepoint of any new entry.

3. **Unicode CLDR / Emoji Charts** — Use
   `https://unicode.org/emoji/charts/full-emoji-list.html` to look up a
   codepoint. Any emoji in block U+1F000–U+1FAFF not yet in the map is
   stripped silently -- add it to the P2/P3 backlog.

4. **Programmatic check**:

```go
// Check if a specific codepoint is substituted, stripped, or preserved:
m := demojify.DefaultReplacements()
r := '\U0001f6a8' // U+1F6A8 police car revolving light
token, substituted := m[string(r)]
stripped := demojify.Demojify(string(r)) == ""
fmt.Printf("substituted=%v token=%q stripped=%v\n", substituted, token, stripped)
```

### How to add a new entry

1. Add the entry to `DefaultReplacements()` in `replacements.go`, in the
   appropriate category comment block (add a new block if needed).
2. Add a test row to `TestDefaultReplacementsEntries` in `replacements_test.go`.
3. Add the codepoint row to the matching section in `docs/replacements.md`.
4. Update the **Category Summary** and **AI Agent Inventory** tables in this
   file: increment entry count, add any new token label, confirm coverage.
5. Update the **Map version** header at the top and the **Last reviewed** date.
6. Run `go test ./...` -- the repo tests enforce that no doc file contains
   live emoji.

### How to update this audit after a batch addition

```
go test ./... -run TestDefaultReplacements  # confirm new entries are tested
go test ./... -run TestRepo                 # confirm docs are emoji-clean
git diff replacements.go | grep '^\+' | grep -oP '"[^"]+"' | sort  # list added keys
```
Then update the entry counts in Category Summary and this file's header.

## Known Gaps

| Issue | Codepoints | Priority | Detail |
|-------|-----------|----------|--------|
| Keycap sequences | digit + U+FE0F + U+20E3 | **High** | `1` + VS16 + U+20E3 sequences generated by Copilot/Claude in numbered step lists; stripped codepoint-by-codepoint but no `1.` substitution. Requires multi-codepoint lookahead in the replace pass. |
| Code fence passthrough | any | Low | Emoji inside Markdown fenced code blocks are not distinguished from prose; no Markdown-aware mode. Acceptable by design; use a preprocessor if needed. |
| Skin-tone modifier sequences | U+1F3FB–U+1F3FF | Low | ZWJ+skin-tone combos are stripped codepoint-by-codepoint; no substitution token. Rare in technical documentation. |
| Flag sequences | U+1F1E0–U+1F1FF pairs; U+1F3F4+tags | Low | Regional indicator and subdivision tag sequences are fully stripped; no substitution defined. Out of scope for text sanitization. |
| Emoji 17.0+ additions | various | Low | Add on demand when observed in generated content. |

## Audit Methodology

1. The source of truth is `DefaultReplacements()` in [`replacements.go`](../../replacements.go).
2. New emoji are added when observed in real-world AI output, READMEs, or gitmoji convention lists.
3. Each new map entry must have a corresponding test row in [`replacements_test.go`](../../replacements_test.go).
4. Token labels follow `[ALLCAPS]` bracket convention. Bare ASCII tokens (`->`, `<3`) are used where the text equivalent is universally understood without brackets.
5. Multiple emoji mapping to the same token (`[HEART]`, `[TOOLS]`) is intentional -- the goal is machine-readable text, not round-trip fidelity.
6. The `TestRepo` suite enforces that all files under `docs/` and the repo root are emoji-clean after `SanitizeFile` -- this acts as a continuous integration gate on the audit itself.
